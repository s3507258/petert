Overview

My project idea would be to create a friendly mobile application for high schools around Australia. The application is mainly catered towards the parents and teachers of high schoolers. I wanted to create an app that lets parents communicate with teachers, check attendance of their kids, check their kids results and find out more information about their kid. Parents would be able to message their kids’ teacher for all classes and be allowed to check daily responses. Rather than the school children telling their parents

Motivation

The reason I wanted to create an app that is catered for the teachers and parents rather than the children is for my parents. According to nces.ed.gov, there are less parents checking up on their kids’ homework and status, the older the kids are. Year 7 students would receive more attention from their parents than a year 12 student. The same for my parents. They would be busy with work and daily activities that they would have no time to check up on my brother. This is why I wanted to create an app that would be easy enough for parents to use and quick enough so that they can check up on it whenever they need to.


Description

The application that I want to create would be for mobile only. It would be on android and IOS. I have done a lot of research and have not found any similar applications aside from university level application like canvas, Moodle. That’s why I wanted to create one for high schoolers and specifically for parents rather than students. When opening the app, it would split into two login details. One for teachers, and one for parents. Both parents and teachers have to sign into their account with a unique id and password. Starting off with the teachers’ main page. The teacher would be able to access five menus. Classes, timetable, Mail, students, and settings.

The first menu that teachers are able to access would be the classes sub menu. Inside the class’s menu, teachers would be able to look at all the classes that they are teaching. If pressed they would be presented with another screen that would show all students that are assigned to the teacher’s classroom along with name, and student id number. When clicking on a student, they are presented their full details and an email button which the teacher can email the student or the parent.

The second icon on the homepage would be the timetable function. Pretty self-explanatory but when the timetable tab is clicked on, it shows the teacher the timetable for the day including the time when classes starts, and the location. There will be an option for the teacher to view classes for individual days up to a week.

The third submenu is mail. This would be the most used section of the entire app. Although a lot of app that integrate mail all have similar functions, the key difference in this is that parents can voice their concerns or thoughts to teachers or vice versa. The main point of this application is that parents and teachers can communicate in a more flexible and suitable way rather than attend parent teacher interviews.

Students is the fourth menu for the teachers app. The student section when clicked on displays all current students that are enrolled into the teachers’ classes. It’ll just show all the students in different sorting orders and when clicked on shows their basic information and contact details. This is an entire section instead of combining into one sub menu due to the fact that it is easier accessible for the teacher.

The last menu is the settings tab. This like most apps, include settings so the user can alter small details like showing the current version, logging out, and swapping users.

The parents home page would be the exact same layout with some key minor differences

·        Parents are allowed to alternate between their different children if they have more than one child.

·        Parents are able to track their kids attendance and see if they go to every class for the  day

·        They are allowed to only contact teachers that are related to the student

·        Another menu which would be news – a section where they can access school letters and important dates

There is not much difference between the teacher and parents besides the ones listed above and maybe more to come with an update/patch.

Tools and Technology

The tools and technology required to start a basic application are

·        android or iPhone for testing purposes

·        A computer/laptop for designing and creating the application

·        A mobile app building program (iPhone) – Appery.io

·        Android app building program – Android Studio

·        Mockup or design programs – user flow diagram

·        Debug system – Android Debug Bridge

·        Icon designer – Angular Material Design

·        A programming language of choice – Kotlin – open source language

·        User interface application to design the layout – React Native

 

Skills Required

The skills that are required to designing and creating an application would be someone that has studied and dabbled with a lot of coding and user interface courses or experience. To design a website one would need sufficient knowledge and time to make and create an application. One would need a team of coders that are specialized to do certain parts. I would choose Java as my language of choice as it is the most common one used to create applications. My user face design would be command line as it is open source and the most common one when used so it’ll be a lot easier to learn with a big community. There would be no special hardware, just only a computer with all the programs listed above. It would be hard to find all these skills within a single person, so hiring a bunch of individuals who have some knowledge in separate parts is a lot easier.

Outcome

The outcome of this project if completed would be very successful and game changing in the high school industry. There are not a lot of applications that are similar to a high school demographic, so if this project can be used as an open source, many high schools can adopt one mainstream app. The original problem will be solved as this project pretty much addresses the concerns of parents who are unable to focus on their children as much due to reasons. This would be a lot more accessible for adults and is friendly enough to be used regularly.
